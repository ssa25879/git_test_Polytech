# 🐣 왕초보를 위한 GitHub .gitignore 사용 설명서

GitHub에 내 코드를 처음 올리시나요? 코드를 올리다 보면 "어? 이 파일은 안 올라가도 되는데..." 혹은 "내 비밀번호가 들어있는 파일이 전 세계에 공개되면 어떡하지?" 하는 걱정이 들 수 있습니다.

이때 사용하는 것이 바로 **.gitignore(깃 이그노어)** 파일입니다. 이 가이드는 여러분의 프로젝트를 깔끔하고 안전하게 지키는 방법을 알려줍니다.

---

## 1. .gitignore가 무엇이고 왜 필요한가요?

이름 그대로 **"Git(깃)이 Ignore(무시)하는 파일들"**의 명단입니다. 이 명단에 파일 이름을 적어두면, 깃이 파일을 추적할 때 해당 파일들은 쏙 빼놓고 관리합니다.



* **저장소 다이어트:** 프로그램 설치 시 생기는 무거운 폴더(`node_modules`, `venv` 등)를 제외하여 저장소 용량을 아낍니다.
* **설정 파일 공유 방지:** 나만 사용하는 에디터 설정이나 운영체제가 자동으로 만드는 불필요한 파일(`.DS_Store`, `Thumbs.db`)이 공유되는 걸 막습니다.
* **보안 강화 (가장 중요!):** 내 개인 비밀번호, API 키, 데이터베이스 주소 등이 담긴 파일이 공개 저장소에 유출되는 것을 원천 차단합니다.

---

## 2. 규칙은 어디에 적나요?

보통은 프로젝트의 최상위 폴더(루트 디렉터리)에 `.gitignore`라는 이름의 파일을 만들어 사용합니다.

| 계층 | 파일 위치 | 특징 및 용도 |
| :--- | :--- | :--- |
| **프로젝트 공통** | 프로젝트 루트 폴더 | **가장 많이 사용.** 팀원 모두에게 동일하게 적용될 규칙. |
| **나만의 규칙** | `.git/info/exclude` | 나만 사용하는 임시 메모장 등을 무시할 때 (GitHub에 안 올라감). |
| **컴퓨터 전체** | 시스템 설정 파일 | 모든 프로젝트에서 공통으로 무시할 파일(OS 생성 파일 등). |

---

## 3. 규칙을 적는 방법 (문법 예시)

파일 안에 아래와 같은 규칙으로 무시할 파일들을 나열하세요.

* `*.log` : 파일 이름과 상관없이 확장자가 `.log`인 파일은 모두 무시
* `temp/` : `temp`라는 이름의 폴더와 그 안의 모든 내용을 무시
* `/config.js` : 현재 폴더에 있는 `config.js`만 무시 (하위 폴더의 파일은 무시 안 함)
* `!important.log` : 로그 파일을 모두 무시하되, `important.log`만은 예외적으로 추적
* `# 주석` : `#`으로 시작하는 줄은 설명글로 처리되어 무시됨

---

## 4. [긴급] 이미 올려버린 파일을 숨기고 싶을 때

이미 GitHub에 올라가 있는 파일을 뒤늦게 `.gitignore`에 적으면 효과가 없습니다. 깃의 '기억(캐시)'에서 해당 파일을 지워줘야 합니다.

**해결 단계 (터미널 입력):**
1.  **캐시 삭제:** 아래 명령어를 입력하세요. (실제 내 컴퓨터의 파일은 삭제되지 않으니 안심하세요!)
    ```bash
    git rm --cached 파일이름
    ```
2.  **변경사항 기록:**
    ```bash
    git add .
    git commit -m "chore: 이제 이 파일은 추적하지 않습니다"
    ```
3.  **반영:** `git push`를 통해 GitHub에 적용합니다.

## 5. 비밀번호와 API 키를 지키는 법

보안을 위해 실제 값이 들어있는 파일은 숨기고, 그 대신 구조만 알려주는 '가짜 파일'을 함께 사용하는 것이 표준입니다.

[Image showing the separation of a sensitive .env file and a template .env.example file]

* **`.env`**: **절대 GitHub에 올리지 마세요.** (명단에 추가 필수!)
* **`.env.example`**: 실제 값 대신 `API_KEY=YOUR_KEY_HERE` 처럼 이름만 적어서 GitHub에 올리세요. 팀원들이 이 파일을 보고 형식을 맞출 수 있습니다.

---

## 6. 직접 쓰기 귀찮다면? (도구 활용)

내가 사용하는 프로그래밍 언어나 운영체제에 맞춰 자동으로 명단을 만들어주는 유용한 사이트가 있습니다.

* **[gitignore.io (Toptal)](https://www.toptal.com/developers/gitignore)**
    * 사이트 접속 후 `Python`, `Visual Studio Code`, `Windows` 등을 검색하여 '생성(Create)'을 누르세요.
    * 생성된 텍스트를 복사해서 내 `.gitignore` 파일에 붙여넣기만 하면 됩니다.
