## 2026-04-06 ~ 07 간 Git 수업 정리

- ### Git 설치 및 사용 준비
  1. [Git](https://git-scm.com/) 에서 Git 최신버전 다운로드

  2. 설치된 _Git Bash_ 터미널을 실행

  3. _Git Bash_ 에서 아래 코드 실행

     ```
     git -v 혹은 git --version          # Git 버전 확인 현재 설치된 버전 -> 2.53.0

     // 최초 설정
     git config --global user.name "Commit시 표기할 이름"
     git config --global user.email "Commit시 표기할 E-Mail"

     git config -l 혹은 git config --list로 설정 확인 가능 (이 때 길이가 길어 $표시가 사라졌을경우 ESC -> :q 로 탈출

     cd <Git 이용할 디렉토리>
     git init
     ```

- ### 기본 Git 명령어
  - git status : 현재 Git 저장소 상태 확인 (수정된 파일, 스테이징 상태, 추적 여부)
  - git add <파일명> / . : 커밋을 위해 스테이징할 파일 선택
  - git commit -m "커밋 메시지" : 스테이징된 변경 사항을 메시지와 함께 저장
      <details>
      <summary>커밋 메시지 작성 요령</summary>
      
          | feat       | 새로운 기능 추가 |
          | fix        | 버그 수정 |
          | docs       | 문서 수정 (예: README.md) |
          | style      | 코드 포맷팅, 공백, 세미콜론 등 (로직 변경 없음) |
          | refactor   | 코드 리팩토링 |
          | test       | 테스트 코드 추가 또는 수정 |
          | build      | 빌드 관련 파일 수정, 의존성 설치/삭제 |
          | chore      | 설정 변경, 기타 자잘한 수정 |
          | ci         | CI/CD 설정 수정 |
          | perf       | 성능 개선 |
          | design     | CSS 등 사용자 UI 디자인 변경 |
          | comment    | 필요한 주석 추가 또는 변경 |
          | rename     | 파일 또는 폴더명 변경, 이동 |
          | remove     | 파일 삭제만 수행한 경우 |

      </details>

  - git log : 커밋 기록 확인
  - git remote add origin "깃허브 주소" : 깃허브 주소를 통해 원격저장소(origin : 기본 별칭) 등록
  - git reset --hard <커밋>: 지정한 커밋 시점으로 파일, 스테이징, 커밋 기록까지 강제 초기화
  - git restore . : 모든 파일을 마지막 커밋 상태로 되돌림
    - git restore --staged . : 스테이징 영역 포함하여 마지막 커밋 상태로 되돌림
  - git clone "깃허브 주소" : 깃허브 주소를 통해 원격저장소에서 로컬저장소로 복제
  - git pull : 현재 원격저장소의 최신 변경 사항을 로컬저장소로 다운로드
  - git push -u origin main/master : 원격저장소(origin으로 등록 한)에 현재 커밋 되어있는 파일을 main/master 브랜치에 업로드
    - -u : 이후 git push만 해도 origin main/master로 자동 연결시킴
